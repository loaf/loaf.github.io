下面，我试着整理一下技术文档，从技术的角度总结一下蒸馏到底是做些什么？怎么做的。

## 蒸馏到底在蒸什么

首先简单来说说，蒸馏（Distill）这个词描述是“学生模型”通过向“教师模型”学习，提升自己的模型能力这一过程。

按“从教师模型那里拿走的是什么”来组织，分成以下7类：

### 1） 只要架构，不管权重

**拿走的是：** 论文里公开的网络结构（attention、MoE 路由、归一化技巧等）。**完全不碰对方的权重、数据、输出。**

* 这是最干净的一类，法律上无争议，因为架构思想不受版权保护（只有具体代码和权重受保护）。
* **现实例子：** DeepSeek的一系列开源技术报告公开了自己的架构，任何人都可以照架构从头训练；Meta 公开 LLaMA 架构也是同理。

### 2）数据生成 / 数据筛选

**拿走的是**：教师模型**生成或筛选的文本数据**，学生用这些数据做普通的 SFT。

其过程相对简单，就是学生向教师问一系列问题，得到一系列的
 
 ```
[(问题, 答案), ...]
``` 
 
 学生模型使用这些数据对自己的模型进行微调（SFT）。
 
 其过程为：
```
教师模型权重 [0.12, -0.34, ...] ← 不公开
              ↓
对 N 条 prompt，让教师生成输出（argmax 文本）
              ↓
收集 [(prompt, output), ...] ← 公开可获取
              ↓
学生做标准 SFT
              ↓
学生学会："遇到这 prompt，往这个方向输出"
```
 
![](https://raw.githubusercontent.com/loaf/sa1/master/blog/images/ChatGPTImage202662315_14_17.png)
 如果按狭义的蒸馏概念^[Hinton,Distilling the Knowledge in a Neural Network, https://arxiv.org/abs/1503.02531]来说，它甚至不算“蒸馏”，就是普通的SFT。

#### 合法性问题
但在OpenAI和Anthropic的服务条款（ToS）中，提到了“禁止用输出开发竞争模型”。
 
 但是在产品ToS中加上这个条款是否合法呢？这是一个法律问题，我无法讨论。
 
 但是有几点是常识：
1.  这只是单方面的条款，不是版权法，不是专利法。即使你点了同意，如果与版权法和专利法冲突，其实是无效的。
2.  输出的内容从法律上到底是谁拥有？
3.  条款是限制生成竞争模型，而不是限定具体技术。
 
根据上面的内容，假定Anthropic的指控属实，多个第三方账号提了几十万个问题，这些问题的数据，从法律上已经属于第三方。然后第三方将数据提供给了MinMax进行微调。也并没有违反上述合同。（不要认为这是狡辩，事实上，当Anthropic发布上述指控后，马上就有国外的网友为国产模型打抱不平，然后将自己的几十万条数据脱敏后上传到Github，主动邀请DeepSeek等公司读取。而Anthropic并无反应。）

事实上，这个条款是厂商用合同去封堵一个版权法本不保护的东西（模型输出的内容），这相当于是变相扩张垄断。

#### 合道德性问题

不能说"不道德"，但确实存在严重的双重标准。

OpenAI和Anthropic用全网版权内容（书籍、网页、代码）训练自己的模型，主张这是合理使用，但却禁止别人用它的输出（无版权数据）来训练自己的模型。

正因为**从法律和道德两方面来说，这个ToS条款都有点心虚，所以争议一直存在，但无论是OpenAI还是Anthropic，都只封号不起诉。**


### 3）推理蒸馏 / CoT Rationale

**拿走的是**：教师模型产生的**推理过程（chain-of-thought / rationale）**，而不只是最终答案。学生学会的是"怎么想"，不只是"答什么"。

这个种类比上述方法更地一步，不仅仅拿到了“问题、答案”对，还拿到了教师模型产生的推理过程。

```
教师模型：对 prompt 不仅给答案，还给出完整推理链
              ↓
收集 [(prompt, "<推理过程> → <最终答案>")]
              ↓
学生在带推理过程的数据上做 SFT
              ↓
学生学会："先这样想，再得答案"（学思路，不只学答案）
```

![](https://raw.githubusercontent.com/loaf/sa1/master/blog/images/ChatGPTImage202662315_18_10.png)

从技术进化的角度来说，它是让小模型能逼近甚至局部超越大模型的关键技术^[Hsieh et al. (Google), *"Distilling Step-by-Step! Outperforming Larger Language Models with Less Training Data and Smaller Model Sizes"*, arXiv:2305.02301]

之所以能拿到推理过程，是因为大模型输出了推理过程。这个“推理过程”是不是有版权呢？很遗憾，“思想和方法”不是版权的内容。

而且讽刺的是，OpenAI以前并不输出推理过程，是被DeepSeek公开推理过程后倒逼着开放的。而且以前有研究显示其推理过程经过简化或者说处理过。反正这家公司和Anthropic类似，整体表现出一个小气齷齪的气质。

### 4）对齐蒸馏 / 行为模仿

**拿走的是**：教师经过 RLHF/DPO 后表现出的**对齐行为**（拒答策略、有用性风格、安全边界）。学生不去复刻教师的"知识"，而是复刻它的"人格/价值观"。

简单来说，就是前面三种方法学到的是“知识”，而这次学到的是老师的“人格”。

现在网上那些玩梗模仿AI的人之所以能让人会心一笑，正是因为这些AI自己的风格有很强的标识度，好比一个真人有自己的语言习惯，性格和处事态度。

具体的实现方法有三种：

**方法1：RLAIF（AI反馈强化学习）**

让一个“已经对齐的教师”评判两段回答哪个更好，生成偏好标签，再用这个标签做RLHF训练学生。
```
一条 prompt
   ├── 学生模型生成回答 A
   └── 学生模型生成回答 B
            ↓
教师按指定的原则进行评判："A 更安全、更有用，因为它解释了边界"
            ↓
得到偏好对 (chosen=A, rejected=B)
            ↓
用偏好对训练一个奖励模型 (Reward Model)
            ↓
用 PPO 等强化学习算法优化学生，让它最大化奖励
```
![](https://raw.githubusercontent.com/loaf/sa1/master/blog/images/ChatGPTImage202662314_10_38.png)

看到上面的过程，看过以前文章的很容易想起Anthropic的宪法AI^[Constitutional AI（Bai et al., Anthropic, arXiv:2212.08073）]，是的，Anthropic正是用这种方法训练自己的模型的。也就是说，这是一个正常的模型训练方法。只不过用法不同而已。

RLAIF用 AI 反馈替代人类反馈——这就把对齐能力变成了可以"批量复制"的东西。一旦教师能自动产偏好，整个对齐链路就被蒸馏出去了。

上图中PPO = Proximal Policy Optimization（近端策略优化）^[由 Schulman et al. 2017 提出,arXiv:1707.06347]，是强化学习里的一类策略梯度（policy gradient）算法，也是 RLHF/RLAIF 走到"用奖励信号优化模型"这一步时，事实上的标准算法。

**方法二：DPO偏好蒸馏**

这是最主流的蒸馏思维链的方法，因为DPO（Direct Preference Optimization）跳过了奖励模型和 PPO，直接用偏好对优化学生。流程更短、更稳定。^[Zephyr,Tunstall et al., arXiv:2310.16944]^[UltraFeedback,Cui et al., arXiv:2310.01377]
```
一条 prompt
   ├── 让多个开源模型生成 4 条回答
   └── 教师（GPT-4）按 1-10 分打分排序
            ↓
取分数最高 vs 分数最低 → 偏好对 (chosen, rejected)
            ↓
DPO 损失：增大 chosen 的似然、压低 rejected 的似然
   L_DPO = -log σ(β·[log π(chosen)/π_ref - log π(rejected)/π_ref])
            ↓
学生权重更新，偏好分布向教师靠拢
```

![](https://raw.githubusercontent.com/loaf/sa1/master/blog/images/image202662313_34_28.png)

DPO把“偏好学习”变成了一个简单的分类损失，不需要训练奖励模型，不需要PPO的复杂采样。

**方法三：直接模仿**

也就是说，从方法上回退到种类二，直接生成数据进行SFT，和数据蒸馏一样，只是目的不同，为了达到这个目的。选择特定的问题即可。


这个种类的蒸馏技术是争议最大的一类，因为对齐能力是闭源厂商最核心的护城河之一，也正是 OpenAI和Anthropic 指控国产模型的主要内容。

但是问题和我在种类二中提到一样， 他们的指控无论从法律还是从道德上都站不住脚。这和你买一本书，从中学到了知识和作者的思维方式有什么区别呢？

### 5）Logits 蒸馏（白盒·输出层）

**拿走的是**：教师对**每个候选 token 的完整概率分布**（logprobs），而不是它最终选择的那个Token。它和数据蒸馏的区别在于，数据蒸馏只拿到答案，而它不仅拿到答案，还拿到了教师选择这个答案的理由。

这其实才是蒸馏这个词在学术上的愿意，由Hinton在2015的经典论文《Distilling the Knowledge in a Neural Network》^[ Hinton, Vinyals, Dean 的 Distilling the Knowledge in a Neural Network（arXiv:1503.02531）]中提出。
```
① 加载教师：BERT-base（白盒，有权重，同进程）
              ↓
② 初始化学生：6 层（教师 12 层的一半），随机初始化
              ↓
③ 对每条训练样本，教师和学生同时前向
              ↓
④ 算三重损失：
   - L_ce   = 软化 logits 的 KL
   - L_mlm  = 掩码 token 交叉熵
   - L_cos  = 隐藏层余弦相似度
              ↓
⑤ 反向传播更新学生（教师冻结）
              ↓
⑥ 得到 DistilBERT：小 40%、快 60%、能力保留 97%
```
![](https://raw.githubusercontent.com/loaf/sa1/master/blog/images/ChatGPTImage202662314_28_39.png)

从技术说明能看出，这需要访问教师模型的内部，国外模型都是闭源，根本做不了。反过来，DeepSeek等开源模型都是可以被国外模型自由蒸馏的。

### 6）中间层表示蒸馏（白盒·特征层）

**拿走的是**：教师**隐藏层的激活向量**、注意力图等内部状态。

这一类**几乎无法通过 API 实现**，必须有权重或在同机部署。所以在"用闭源 API 蒸馏竞争对手"的语境下，这一类基本不适用——它更多用于**自家大模型蒸馏到自家小模型**（如 DistilBERT、MiniCPM、Qwen 系列内部蒸馏）。

整个领域的起点是 Romero et al. 的 FitNets: Hints for Thin Deep Nets（ICLR 2015, arXiv:1412.6550）。

```
① 加载教师：白盒，有权重，同进程
              ↓
② 设计学生：比教师深但薄（层数↑，维度↓）
              ↓
③ 选 hint/guided 层对：
   教师 hint：第 K 层（如第 16 层）
   学生 guided：第 K' 层（如第 20 层）
              ↓
④ 构建投影层 regressor（W_r：学生维 → 教师维）
              ↓
⑤ 阶段 1 - Warm-up：
   只用 hint 损失 = MSE(W_r · h_student, h_teacher)
   训练学生中间层 + regressor
              ↓
⑥ 阶段 2 - Joint：
   切换到 logits 蒸馏损失（Hinton KL）
   在中间表示对齐基础上对齐输出
              ↓
⑦ 推理时丢掉 regressor，只用学生
              ↓
⑧ 得到比教师深、薄、快，但能力接近的学生
```

![](https://raw.githubusercontent.com/loaf/sa1/master/blog/images/ChatGPTImage202662314_44_52.png)


### 7） Speculative Decoding（推理时协作）

**拿走的是**：什么都不"拿"——教师模型**始终在线**，由一个小模型（draft model）先猜若干 token，教师只验证/修正。这是**推理加速**而非训练蒸馏，但常被并列讨论。

教师在这里是主动参与者，通常是你有权使用的模型（自家大模型或开源权重），所以法律上毫无争议。

传统意义上，这一类并不是“蒸馏”，只不过，它同样涉及到"学生模型 + 教师模型"的组合，所以经常并列讨论。

为什么要这么做呢？主要是大模型推理成本过高，与其让大模型干所有的活，不如把开头的一些简单的工作交给学生模型，然后教师审核一下，继续后面有难度的工作。

```
① 离线准备：
   - 教师模型 T（大，如 Llama-70B）
   - 小工模型 D（小，如 Llama-7B），通常用类别⑤/⑥蒸馏得到
              ↓
② 在线推理（对每个请求）：
   while 还没生成完：
      a. 小工 D 快速串行猜 K 个 token：x₁, x₂, ..., xₖ
              ↓
      b. 教师T 对 [已有上下文 + x₁..xₖ] 做一次前向
         同时得到每个位置的 logits：p₁, p₂, ..., pₖ
              ↓
      c. 逐位置验证（带概率修正）：
         对 i = 1..K：
           接受概率 = min(1, p_i(xᵢ) / q_i(xᵢ))
           if rand() < 接受概率: 接受 xᵢ
           else: 拒绝，从残差分布采样 x' ，break
              ↓
      d. 如果全部 K 个都接受：免费多送 1 个（教师第 K+1 位置的输出）
              ↓
      e. 回到 a，继续猜下一批
              ↓
③ 结果：输出分布 = 教师 T 的分布（数学保证），速度提升 2~3×
```

![](https://raw.githubusercontent.com/loaf/sa1/master/blog/images/ChatGPTImage202662315_04_11.png)


## 终结蒸馏叙事

也就是说，蒸馏是有效的学习方法之一，但指望使用蒸馏来偷到前沿大模型的能力，基本上是痴心妄想。

要不然的话，欧洲怎么蒸不出自己的大模型？日本怎么蒸不出自己的大模型？印度怎么不去蒸自己的大模型？

在蒸馏技术的工程实践中，对学生模型的要求其实比想象中要高的多，你是一个弱智，让院士来教也是白搭。国产大模型的地位也反过来证明了他们的实力不是通过污蔑盗窃就能抹杀的。

本质上，和你买一本书来自学没有什么区别，能学到多少在于个人能力和态度而不在于阅读这种行为。

每天重复炒作中国大模型通过“蒸馏”抄袭美国大模型的基本上都是坏人，而鹦鹉学话的则大多是蠢人。和这几天印度人天天在炒作“士农工商”是中国的种姓制度一样，不过是坏人和傻子之间的“自激振荡”。

终结这种污蔑的方法和问题其实也很明确：

![](https://raw.githubusercontent.com/loaf/sa1/master/blog/images/20260624214540183.png)

但大家也都清楚，当它们无法用蒸馏叙事来纠缠后，虫豸们会找到一个新话题来继续抹黑。
